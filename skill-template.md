# Skill scaffolding template

When generating a SKILL.md from an approved candidate, use this template. Populate every bracketed field from the candidate's synthesis fields (Phase 4 of skill-finder) plus any reference artifacts found during discovery.

## Template

```markdown
---
name: [skill-name from candidate]
description: [Pushy description, ~50-80 words. Include what it does + specific contexts/phrases that should trigger it. Bias toward triggering — combat undertriggering by being explicit about when to use this skill, even if the user doesn't use the literal skill name.]
---

# [Skill Name]

[One-paragraph overview of what this skill does and why it exists. Pull from the candidate's "What it does" line and expand with the underlying pattern evidence — what manual workflow this replaces.]

## When to use

Trigger this skill when:
- [Primary trigger phrase from candidate]
- [Secondary trigger phrase]
- [Edge cases or related contexts]

Do NOT use this skill when:
- [Out-of-scope cases]

## Inputs

This skill expects:
- [Input 1 from candidate "Inputs needed each run"]
- [Input 2]
- [...]

If any required inputs are missing, ask the user before proceeding.

## Workflow

[Numbered steps. Pull the structural template from the discovery artifacts — if the user has 6 monthly investor updates with similar shape, the workflow steps mirror that shape.]

1. [Step 1]
2. [Step 2]
3. [Step 3]

## Output

[Describe the output format, mirroring what the user actually produces today. Reference any template files in references/ that the skill should follow.]

## Voice and conventions

[If the discovery artifacts revealed a consistent voice or set of conventions — e.g., "always opens with a one-line headline metric" or "uses 'we' not 'I'" — capture them here. This is where skill-finder adds real value over a generic template: the user's voice gets preserved.]

## Reference files

- `references/template.md` — The structural template extracted from past artifacts
- `references/voice-examples.md` — Sample passages illustrating tone and conventions

<!-- Generated by skill-finder on [DATE]. Review before installing. Voice and conventions extracted from [N] sample artifacts in [source paths]. -->
```

## Reference file generation

For skills with strong template signals, also generate `references/template.md` containing the structural pattern extracted from discovery (with PII/specific data redacted to placeholders).

For skills with strong voice signals (recurring email patterns, recurring exec comms), generate `references/voice-examples.md` with 2–3 anonymized excerpts plus a one-paragraph distillation of the voice.

## Final check before writing

Before writing the SKILL.md file, verify:

1. The description is "pushy" — explicit about triggering even when user doesn't use the literal skill name
2. The workflow steps mirror the actual structural pattern from the user's artifacts, not a generic template
3. The voice/conventions section captures real observations, not platitudes
4. The generated note at the bottom records the source artifacts for traceability
